## Synchoronized는 제대로 알고 써야한다.
웹 기반 시스템을 개발할 때 여러분들이 스레드를 컨트롤할 일은 별로 없다.

우리가 개발하는 WAS는 여러 개의 스레드가 동작하도록 되어있다. 그래서 synchoronized를 자주 사용한다. 스레드가 어떻게 작동되는지 알아보자.

### 프로세스와 스레드
클래스를 하나 수행하거나 WAS를 기동하면, 서버에 자바 프로세스가 하나 생성된다.
<br>하나가 생성되는지 여러개가 생성되는지는 윈도의 자원관리자나 유닉스의 프로세스를 조회해보면 된다.
<Br>하나의 포로세스에는 여러개의 스레드가 생성되고, 단일 스레드에는 생성되어 종료될 수 있고, 여러개의 스레드가 생성되어 수행될 수 있다. <br>프로세스와 스레드의 관계는 1:다 라고 보면된다.

왜 이런 관계가 만들어질까? 스레드는 다른 말로 LWP라고도 한다. 가벼운 프로세스이고, 프로세스에서 만들어<br>
사용하고 있는 메모리를 공유한다. 즉 프로세스가 하나씩 뜨는 것보다는 성능이나 자원 사용에 있어서 많은 도움이 된다.

### Thread 클래스 상속과 Runnable 인터페이스 구현
자바는 다중 상속을 지원하지않는다. 스레드를 사용해야 할 때 이미 상속받은 클래스가 존재하면 Runnable인터페이스를 
<br>따로 구현해야한다. 아주 간단한 스레드 관련 클래스 두개를 보자 하나는 Runnable인터페이스를 구현한 클래스이다.
<br>두번째는 Thread클래스를 확장한 경우다. 

Thread를 상속받은 경우는 Start메서드를 호출하면된다. 하지만 Runnable인터페이스를 구현한 경우 Thread클래스의 Runnable인터페이스를 매개변수로 받는 생성자를 사용해서 Thread클래스를 만든 후 Start 메서드를 호출해야한다 .

그렇게 하지말고 그냥 run()메서드를 호출하면 새로운 스레드가 생성되지 않는다.

실행해보면 알지만, 결과값이 고정되지않는다. 우선순위를 지정하지 않았으므로 Thread클래스를 상속받은 결과가 먼저 나올 수 있고, Runnable인터페이스를 구현한 결과가 먼저 나올 수 있다.

### sleep(), wait(), join()메서드 
현재 진행중인 스레드를 대기하도록 하기 위해서는 세가지 메서드가 필요하다. wait은 Object가 부모라서 어떤 클래스에서든지 사용가능하다.

sleep()은 명시된 ms만큼 해당 스레드가 대기한다. static 메서드이기 때문에 반드시 스레드 객체를 통하지 않아도 사용할 수 있다.

join()은 명시된 메서드가 끝날때 까지 기다린다. 매개변수를 지정하지 않으면 죽을 때까지 계속 대기한다.

### interrupt(),notify(), notifyAll() 메서드
앞서 명시한 세개의 메서드를 모두 멈출 수 있는건, interrupt이다. 호출이 되면 중지된 스레드에서는 InterruptedException이 발생한다. 제대로 수행됐는지 확인하려면 interrupted()메서드를 호출하거나 isInterrupted()를 호출하면된다.

두개의 차이는, 상태변경 시키는것과, 단지 스레드의 상태만을 리턴하는것 두개이다.

isAlive()메서드는 살아있는지 확인한다.

notify() 메서드와 notifyAll 메서드는 모두 wait메서드를 멈추기 위한 메서드다. 이 두 메서드는 오브젝트 정의이고, 대기상태로 변경된 스레드를 깨워준다.

### interrupt메서드는 절대적인것이 아니다. 
이 메서드는 해당 스레드가 'block'이 되거나 특정상태에서만 동작한다. 

### synchronized를 이해하자.
스레드 관련중에 가장 많이 사용하는것은 이거일것이다. 제대로 사용하지못하면 성능문제를 야기할 수 있다. <br>
동시에 일어난다. 동시에 진행한다는 뜻이다. <br>

> 뭐가 동시에 일어날까? <br>
무한도전 프로그램이 있다. 초창기에는 PD가 밥을 주지 않아서 그랬는지, 항상 먹는것 때문에 싸웠다. 매번 6명이서 달려들다가 먹지도 못하고 땅에 떨어뜨리기 일쑤였다. 만약 누가 6등분 했다면 먹었을것이다.

> 자바도 마찬가지다. 하나 객체에 여러 요청이 동시에 달려들면 원하는 처리를 하지도 못하고 이상한 결과가 나올 수 있다. 그래서 synchronized를 사용하는것이다. 이 식별자를 사용하면 천천히 한명씩들어와! 라는 뜻이다. 해당 블록이나 메서드에서 제어하게 된다.

언제 동기화를 해야할까? 
- 하나의 객체에서 여러 스레드 동시 요청일 경우
- static으로 선언한 객체를 여러 스레드에서 동시에 사용할 경우다.

1. 여러 기부자가 어떤 기부금을 처리하는 단체에 기부금을 내야한다. 기부금을 내는 사람은 스레드로 구현되며, 기부금을 내는 사람의 이름 정보가 있어야한다. 기부금을 받는 단체는 기부금을 받을 창구로 donate()라는 메서드를 제공한다.
기부한 전체 기부금을 확인하는 메서드는 getTotal()이다.

### JVM내에서 synchronization은 어떻게 동작할까?
자바의 Hostpot VM은 자바모니터를 제공하므로써 스레드들이 상호 배제 프로토콜에 참여할 수 있도록 돕는다.<br>
자바 모니터는 잠긴 상태나 풀림중 하나고 동일한 모니터에 진입한 여러 스레드들 중에 한 시점에는 단 하나의 스레드만 모니터를 가질 수 있다. 

이 상태가 되면 스레드는 많은 비용이 드는 인스트럭션 재배열 작업을 통해서 잠김과 풀림 작업을 수행할 수 있게된다.
<br>이 작업들은 진보된 적응 스피닝 기술을 사용하여 처리량을 개선시킬 수 있다고 한다. 결과적으로 동기화 성능은 보다 빨려졌다.
